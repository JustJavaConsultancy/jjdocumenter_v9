About Our System

Our system is designed to support graduates in requesting their academic transcripts and certificates while enabling institutional staff to process these requests efficiently. The platform ensures compliance with Nigerian regulations and provides a secure environment for all users.

With our system, graduates can easily submit requests, make payments, and track the status of their requests. Institutional staff can validate requests, generate secure documents, and manage system settings. The system also provides auditors with access to reports and logs for transparency and accountability.

Key Features

Alumni Portal

Graduates can request academic transcripts and certificates, make payments, track the status of their requests, and download approved documents.

Staff Console

Institutional staff can validate academic records, approve or reject requests, and generate secure documents.

Secure Document Management

The system ensures that all documents are securely stored and can be accessed only by authorized users. Graduates can also verify the authenticity of their documents via a public verification page or API.

Additional Information

Technology Stack

  • Backend Frameworks: Spring Boot for building RESTful services and handling business logic.
  • Frontend Frameworks: Thymeleaf for server-side rendering and HTMX for dynamic content loading.
  • Databases: PostgreSQL for relational data storage and MongoDB with GridFS for document management.
  • Realtime Data transfer: WebSockets for real-time notifications.
  • Messaging/Event Streaming: RabbitMQ for asynchronous communication and event handling.
  • Authentication/Authorization: Keycloak for identity management and OAuth2.1 for secure access.
  • Testing Framework: Cucumber for behavior-driven testing and JUnit for unit tests.
  • Document Management: GridFS for storing documents and MinIO for object storage.
  • Deployment Options: Docker for containerization and GitHub Actions for CI/CD.

Compliance and Security

The system is designed to comply with Nigerian regulations, including the Nigeria Data Protection Regulation (NDPR). It implements robust authentication and encryption to ensure the security of user data and documents.

The system also provides role-based access control (RBAC) to ensure that users can only access the information and features that are relevant to their roles.
